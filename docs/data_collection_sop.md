# Product Image Collection SOP

Recognition quality is set by image quality, not by code. For the RESVI
catalogue (much of it generic, unbranded, look-alike goods) this is the single
biggest lever. Whoever enrolls products should follow this.

## Per product
- **Minimum 5 images**, more for look-alike items (plain plastic, steel, pooja
  items). 8–10 is better for anything that resembles other stock.
- **Plain background**, ideally white. A white sheet or board on the counter.
- **Even lighting**, no harsh shadows, no glare/reflection on glossy surfaces.
- **Object fills the centre**, framed like the scan guide box.
- **Multiple angles**: front, slight left, slight right, top, and any side with
  distinctive text/markings.
- **No hands, no clutter** in frame.

## Colours
- Same item, multiple colours, **same selling price** → ONE product. Add all
  colour images under it. Do not create a separate product per colour.

## Sizes (critical)
- Same item in different sizes → **separate products that share a family key**.
  The camera cannot judge absolute size from one frame, so the app will ask the
  cashier to pick the size. Name them clearly ("Bucket 5L", "Bucket 10L") and
  give them the same family so the app groups them in the picker.

## After capture
- Saving the product enrols it immediately — no training step, no waiting.
- If recognition for an item is weak in daily use, add 2–3 more images of it
  from real counter conditions. It improves on the next scan.
